Durable cross-project working preferences:

- **Quality first** — correctness over speed; understand → test → document → review → profile.
- **Test-driven** — write the failing test first for accuracy-critical paths.
- **No emojis** in code, commits, comments, or docs; precise technical language.
- **Conventional Commits** (`feat|fix|docs|refactor|test|chore|perf|build|ci`).
- **Never commit or push unless explicitly asked.**
- **Read before write** — see [[feedback-read-before-write]].
- **Curate, don't dump** — see [[feedback-curate-not-dump]].
