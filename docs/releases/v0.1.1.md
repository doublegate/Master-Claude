# Master-Claude v0.1.1

A patch release applying fixes from automated code review (gemini-code-assist and GitHub
Copilot) on the v0.1.0 release PR. No functional changes to the tooling or knowledge base.

## Fixed

- **Release workflow — robust manual dispatch.** `release.yml` now normalizes the
  `workflow_dispatch` `tag` input: a pasted `refs/tags/v0.1.0` (a common GitHub UI pattern) has
  its prefix stripped, and the value is validated against the `v[0-9]*` shape before use. Manual
  releases no longer fail on that input. Tag-push releases are unaffected.

## Changed

- **Changelog accuracy.** The release-workflow and README-badge entries — which shipped in the
  `v0.1.0` tag — are now attributed to the `[0.1.0]` section instead of `[Unreleased]`, so the
  changelog reflects what each tag actually contains.

## Verification

- `shellcheck --severity=warning bin/*.sh test/run.sh` clean.
- `sh test/run.sh` passes (18 sandboxed assertions).
- Both workflow YAMLs parse; no instruction module exceeds 200 lines.

## Artifacts

- `master-claude-v0.1.1.tar.gz` — the full installable system (tracked files at the tag).
- `master-claude-v0.1.1.tar.gz.sha256` — checksum for verification.

## Upgrading

No action required. v0.1.1 is drop-in; the only behavioral change is in the release workflow's
manual-dispatch path. See [`docs/releases/v0.1.0.md`](v0.1.0.md) for the full feature set.
